# RunPod 高速画像生成環境 取扱説明書 (マニュアル)

このドキュメントは、提供されたファイル群を使用して RunPod 上に ComfyUI の高速画像生成環境を構築し、バッチ処理から自動シャットダウンまでを自動化するための手順をまとめたものです。

## 全体の流れ

1. **Docker イメージのビルドと公開**：あらかじめ ComfyUI やカスタムノードを含んだコンテナイメージを用意します。
2. **RunPod インスタンスのデプロイ**：作成したイメージを使用して、RunPod 上でインスタンス（Pod）を立ち上げます。
3. **APIキー（RunPod Secrets）の準備**：HuggingFace や RunPod の API キー、Google Drive の認証情報を設定します。
4. **セットアップノートブックの実行**：`runpod_setup.ipynb` を実行し、モデルのダウンロードから画像生成、Google Drive への自動転送、自動停止までを全自動で行います。

---

## ステップ 1: Docker イメージの準備

今回提供されている Dockerfile には、Python や `uv` (高速パッケージマネージャー)、各種 ComfyUI 拡張ノード、自動監視スクリプトが含まれています。

1. コマンドプロンプトまたはターミナルを開き、`runpod_workflow_plan.md` のあるディレクトリ (`c:\AIwork\ImageGeneration\Runpod\Runpod_ComfyUI`) に移動します。
2. イメージをビルドし、Docker Hub や GitHub Container Registry にプッシュします。（例：ご自身の DockerHub ユーザー名が `yourname` の場合）
   ```bash
   docker build -f docker/Dockerfile -t yourname/runpod-comfyui:latest .
   docker push yourname/runpod-comfyui:latest
   ```
   ※事前に Docker Hub のアカウント作成と `docker login` が必要です。

---

## ステップ 2: RunPod 環境変数の登録 (推奨)

ノートブック上にベタ打ちすることも可能ですが、セキュリティ上 RunPod の「Secrets」機能を利用して API キーを安全に渡すことを推奨します。

1. RunPod のダッシュボードの左メニュー「Settings」から「Secrets」を開きます。
2. 以下の変数名で値を登録します：
   * `MY_RUNPOD_API_KEY`: RunPod の API キー (作成したインスタンスを自爆形式で自動終了させるために必要です)
   * `HF_TOKEN`: HuggingFace の Access Token (モデルダウンロード用)
   * `RCLONE_CONF_B64`: Google Drive やクラウドストレージに自動アップロードするための RClone の設定を Base64 エンコードしたもの
     - **生成方法**: コマンドラインなどで `base64 -w0 ~/.config/rclone/rclone.conf` を実行した結果の長大な文字列

## ステップ 3: RunPod インスタンスの起動

1. RunPod の「Deploy」画面から、**RTX 4090** (または 5090) などの高性能 GPU インスタンスを選択します。
2. **Template** にて「Customize」または自身で作成したテンプレートを選択し、**Container Image** に ステップ 1 で作成したご自身の Docker Hub イメージ名（`yourname/runpod-comfyui:latest` など）を入力します。
   **Expose HTTP Ports** に `8188,8888` をカンマ区切りで入力します（8888を省略するとJupyterLabボタンが表示されません）。
3. **Storage Configuration** を以下の通り設定します（課金最小化のため永続ストレージは使用しない設計）。

   | 項目 | 設定値 | 備考 |
   |---|---|---|
   | **Network Volume** | **0 GB** | 未使用時の課金をゼロにするため使用しない |
   | **Container Disk** | **50 GB** | モデルDL一時領域 + 生成画像バッファ用 |

   > Container Disk の内訳目安: ComfyUI本体・依存関係 (~5 GB) + モデルファイル UNET/VAE/CLIP/LoRA (~15 GB) + 生成画像一時バッファ (~10 GB) + 余裕 (~20 GB)
   > モデルは毎回起動時に HuggingFace から高速DLし、生成画像は随時 Google Drive へ転送するため Network Volume は不要です。
4. **Deploy** をクリックして起動します。

---

## ステップ 4: JupyterLab 上でのプロンプトバッチ実行

Pod が「Running」になり、「Connect」ボタンを押して「Jupyter Lab」を起動します。

1. Jupyter 画面左側のファイルツリーに **`runpod_setup.ipynb`** をアップロードします。（Jupyter 上の `/workspace` 直下）
2. アップロードした `runpod_setup.ipynb` をダブルクリックして開きます。
3. **ノートブックの実行**:
   * **Cell 0**: RunPod の Secrets を設定している場合は空欄のままで実行します。設定していない場合はセル内にキーを直接入力して実行してください。
   * **Cell 1**: ワークフローの設定を行います。使用するモデル (`turbo` か `zimage`)、モデルの重み(`WEIGHT_DTYPE`)、アスペクト比(`ASPECT_RATIO`)、およびバッチサイズ (`BATCH_SIZE`) などを設定して実行します。
     ※アスペクト比は `'Instagram Portrait - 1080x1350'`, `'1:1 Square - 1080x1080'`, `'Instagram Story - 1080x1920'`, `'Instagram Landscape - 1080x566'` などの一覧からコピペして指定可能です。
   * **Cell 2**: 環境の初期化を行います。実行すると、自動的に左のファイルツリーの直下に `runpod_prompts.txt` というダミーのプロンプトファイルが自動生成されます。
   * **追加作業**: 事前にローカルのツールで生成した `runpod_prompts.txt` がある場合は、`runpod_setup.ipynb` と一緒にそのままファイルツリーにドラッグ＆ドロップして上書き（アップロード）してください。
     *(書式例)*
     ```text
     positive: 1girl, smiling
     negative: nsfw, text
     ---
     positive: 1boy, running
     negative: worst quality
     ```
     ※zimageのデフォルトネガティブを使用する場合は、`negative:`行を省略することも可能です。
   * **Cell 3**: 約数GB〜十数GBのモデルを数十秒〜数分で超高速ダウンロードします。
   * **Cell 4**: Google Drive への接続テストを行い、指定した LoRA モデルを自動ダウンロード、および `auto_uploader.py` をバックグラウンドで開始します。
   * **Cell 5**: ComfyUI 本体を高速モード (`--fast` / `--bf16-vae` 等) で起動します。
   * **Cell 6**: Nodeの設定や読み込んだプロンプト情報に基づき、ComfyUIへ生成ジョブを一気に送信（キューイング）します。
   * **Cell 7**: キューの消化とGoogle Driveへのアップロード完了を監視し、すべて終わったら**自動的にRunPodインスタンスを終了（破棄）**します。

これで手動でのシャットダウン忘れによる高額請求を予防しつつ、最短・最速コストで大量の画像を生成・回収することが可能です。
